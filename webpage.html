Exploring Git Commands through Collaborative Coding

1. Setup and Initialization

Install and configure Git:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Create a new repository:

git init collaborative-project
cd collaborative-project

2. Creating and Committing Files

Create a new file and commit changes:

git add .
git commit -m "Initial commit with project setup"

3. Working with Remote Repositories

Connect to a remote repository and push changes:

git remote add origin <repository_url>
git push -u origin main

4. Branching and Feature Development

Create and switch to a new branch:

git branch feature-xyz
git checkout feature-xyz

5. Collaboration and Pull Requests

Push a new branch and create a pull request:

git push origin feature-xyz

6. Handling Merge Conflicts

Resolve conflicts and commit:

git add <conflicted_file>
git commit -m "Resolved merge conflict"

7. Synchronization and Updates

Pull the latest changes from the main branch:

git pull origin main

8. Best Practices

  • Write meaningful commit messages.
  • Use .gitignore to exclude unnecessary files.
  • Work on feature branches instead of main.
  • Review code before merging.
